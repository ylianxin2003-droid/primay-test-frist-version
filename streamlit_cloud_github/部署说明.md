# GitHub + Streamlit Community Cloud 部署指南

## 部署要求

- GitHub 入口：`streamlit_cloud_github/app.py`
- Python：3.11
- 数据源：SERENE `/api/download-output/`、`/api/download-forecast/` 和 Kp/ap
- 科学解释器：`breid-phys/aida-ionosphere` v0.1.3（MIT）

因为 Streamlit Cloud 不能给现有应用原地更换 Python，请先保存 URL 和
Secrets，删除旧应用，再创建应用并在 **Advanced settings** 选择 Python
3.11。

## Secrets

```toml
SERENE_API_BASE_URL = "https://spaceweather.bham.ac.uk"
SERENE_API_TOKEN = "新生成的 API token"
SERENE_API_TIMEOUT = "30"
SERENE_AUTH_SCHEME = "Token"
```

不要上传 `.env` 或 `.streamlit/secrets.toml`。聊天中曾公开的 token 必须撤销。

## 工作方式

一次 API 请求下载一个时间的 raw AIDA 状态文件。官方 AIDA 解释器在本地
计算所有经纬度点，因此地图点数和 API 请求次数相互独立。Kp/ap 是全球
指标，不进入区域地图。

ICAO-style 页面会读取分析时刻前3小时的37个五分钟状态、30个每日PSD
参考状态，以及官方 +3h/+6h forecast。首次加载可能较慢；相同产品会由
有界缓存复用。S4、sigma-phi、PCA和SWF不属于当前SERENE AIDA输出，页面
必须显示 `Not available from SERENE`，不能显示虚构的0或OK。

## 常见问题

| 问题 | 处理 |
|---|---|
| 401/403 | 检查新 token 和 `Token` 认证方式 |
| 返回 non-HDF5 | 检查 base URL 必须是 `spaceweather.bham.ac.uk` |
| 导入 aida 失败 | 确认 Python 3.11，查看依赖安装日志 |
| 选定时间没有输出 | 换到已有归档时间；历史时间使用 final 产品 |
| Kp 没有区域色块 | 正确行为：Kp/ap 仅为全球背景指标 |
| PSD 显示 N/A | 选定时刻之前可能没有完整30天AIDA归档、部分参考状态不可用，或缺少完整96小时Kp历史 |
| +3h/+6h 显示 N/A | SERENE 没有该分析时刻对应的官方 forecast；观测数据仍可使用 |

项目仅作学术演示，不是正式 ICAO 业务预警系统。
