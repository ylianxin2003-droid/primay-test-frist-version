# GitHub + Streamlit Cloud 部署指南

## 文件作用

| 文件 | 作用 |
|---|---|
| `app.py` | Streamlit 页面入口和交互控制 |
| `config.py` | 读取本地 `.env` 或云端 Secrets |
| `serene_client.py` | AIDA 输出目录、HDF5 和 Kp/ap 网络读取 |
| `aida_grid.py` | 验证 HDF5，并按指定边界框和间距在内存取点 |
| `data_loader.py` | 每个输出下载一次、去重并合并统一数据 |
| `alert_engine.py` | 研究原型告警规则 |
| `forecast_engine.py` | 趋势/持续性短时预测 |
| `visualisation.py` | 图表和参数地图 |

不要上传 `.env` 或 `.streamlit/secrets.toml`，它们可能包含真实 token。

## 部署步骤

1. 把本文件夹内容推送到 GitHub 仓库。
2. 在 [Streamlit Community Cloud](https://share.streamlit.io) 创建应用。
3. 主文件选择 `app.py`（若仓库外面还有一层目录，则选择 `streamlit_cloud_github/app.py`）。
4. 在 **Settings -> Secrets** 填写：

```toml
SERENE_API_BASE_URL = "https://serene.bham.ac.uk"
SERENE_API_TOKEN = "你的 API token"
SERENE_API_TIMEOUT = "30"
SERENE_AUTH_SCHEME = "Token"
```

5. 部署或重启应用。

## 常见问题

| 问题 | 处理 |
|---|---|
| 找不到 Python 模块 | 确认 `requirements.txt` 与 `app.py` 位于部署目录 |
| 返回 SERENE 登录页 | Chrome 登录不能用于云端；检查 API token 是否有输出目录和下载权限 |
| 选定时间没有产品 | 当前时间使用 ultra，历史时间使用 final；选择更接近归档产品的时间 |
| 全球地图点很多 | 增大 grid step 可减少内存计算和绘图点数，但 AIDA 下载次数不变 |
| Kp 没有区域色块 | 这是正确行为；Kp/ap 是全球背景指数，不是区域地图变量 |

应用仅作学术演示，不是正式 ICAO 业务预警系统。
